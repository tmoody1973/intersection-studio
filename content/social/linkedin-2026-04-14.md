The Studio Dashboard now has a CEO that delegates.

You write a goal. The CEO agent reads it, breaks it into tasks, and routes each task to the right agent automatically. Engineering research goes to the eng lead. Brand direction goes to the creative director. Content summary goes to the content writer.

That's Paperclip-style delegation — borrowed from Paperclip AI's orchestration model — applied to a one-person studio. I'm not the manager. The CEO agent is. I'm the architect who approves before anything ships.

Also shipped this week: direct agent chat. Every agent in the org chart is now directly conversable from the dashboard. You click the creative director, you talk to the creative director. The full project context — goals, thread history, recent tasks — gets injected before the first message. The agent already knows what you're working on.

Every conversation is logged to the project thread as institutional memory. The agents remember. Next time someone works that project, it starts with context, not from zero.

Stack: Next.js + Convex + Hermes on Fly.io + OpenRouter for model routing across 157 models.

Two active projects: MKE Neighborhood Dashboard (civic data for Milwaukee's northside) and the Studio Dashboard itself. Different problems, same methodology — research first, build from documents, ship when the spec is clear.

The operating model is: Hermes agents draft. Humans approve. Claude Code builds. Knowledge compounds.
