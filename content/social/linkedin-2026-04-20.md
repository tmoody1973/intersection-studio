# LinkedIn — 2026-04-20

The studio's agent interaction model changed last week. Not incrementally — structurally.

Before: fire a task, wait 30-60 seconds, hope the callback returns something useful. The agent was a black box. You'd submit a goal and stare at a spinner.

Now: you're in the session. You can watch the reasoning, steer mid-task, and see exactly what context the agent loaded before it responded.

We call it Co-Work Mode. It runs on CopilotKit + AG-UI protocol, connected to our Mastra agent backend. What's in the build:

- 40/60 split view: your conversation on the left, agent's live work product on the right
- Brain cards showing what context each agent loaded before responding
- Session timeline scrubber — replay any completed session step by step
- Cost ticker so you know what each session actually costs
- Sound design via Web Audio API — audio cues for start, complete, error
- Smart copy that structures the agent output as a handoff prompt for Claude Code
- Agent status bar with elapsed time + escalating feedback ("still working... this is taking longer than expected")

The backend shifted too. From 12 Hermes profiles using fire-and-callback to 5 Mastra agents with AG-UI streaming. Fire-and-forget is now the fallback, not the default.

Dev Diary #019 — "The Agents Learn to Talk" — documents what we built, what we traded, and why this is the right direction for a studio building cultural products.

Real question: is watching agents work in real time actually useful, or is it just satisfying? Trying to figure out if Co-Work Mode changes how decisions get made, or if it's mostly about trust-building.
